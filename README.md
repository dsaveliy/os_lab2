# Многопоточное умножение матриц

Лаба реализует **блочное умножение матриц** с использованием трёх различных API для работы с потоками:
- **std::thread** (C++ на Windows, UNIX)
- **WinAPI** (Windows)
- **pthread** (UNIX)

## Структура

| файл                 | назначение                                            |
|----------------------|-------------------------------------------------------|
| `matrix.h/cpp`       | стандартные операции для матриц                       |
| `stdthread.h/cpp`    | потоки std::thread                                    |
| `winapithread.h/cpp` | потоки WinAPI                                         |
| `pthread.h/cpp`      | POSIX потоки                                          |
| `main.cpp`           | тестирование                                          |

## Вывод по резльутатам

Из полученных данных видно, что на очень маленьких матрицах многопоточность не даёт выигрыша и даже проигрывает классическому подходу из-за оверхеда на создание и управление потоками. На средних и больших размерах матриц ускорение появляется, но только если количество потоков подобрано оптимально и не слишком большое по сравнению с числом процессорных ядер. Если потоков становится слишком много, то время работы снова начинает расти, поскольку система тратит больше ресурсов на переключение между потоками, чем на реальную работу. При сравнении всех трёх реализаций получаем, что std::thread, WinAPI и pthread показывают практически одинаковую производительность и различаются незначительно.
